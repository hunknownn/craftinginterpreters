1왜 8장인가?

7장 인터프리터는 표현식 하나를 계산하고 끝났다.

7장까지

1 + 2 * 3   // → 7

값 하나를 계산하면 끝. 기억도, 순서도 없음.

8장부터

var a = 1;   // 기억
print a;      // 출력
a = a + 1;  // 변경
print a;

순서대로 실행 + 값을 기억 → 진짜 프로그램.

두 가지를 추가한다 — Statements(효과를 내는 실행 단위)와 State(변수에 담기는 상태).

2표현식 vs 문장

표현식 (Expression)

값을 만든다

"계산하면 뭐가 나와?"
2+3 → 5

인터프리터: evaluate() → 값 반환

↔

문장 (Statement)

효과를 낸다

"실행하면 뭐가 일어나?"
print 5; → 화면 출력

인터프리터: execute() → 반환 없음(Void)

문장은 보통 안에 표현식을 품는다:  print (2 + 3); — 문장이 표현식을 재료로 쓴다.

3문장 추가하기

두 종류로 출발: 표현식 문장 + print 문

표현식 문장 exprStmt

brunch("eggs");

표현식 + ;. 값보다 부수 효과(함수 호출 등)를 위해.

print 문 printStmt

print 2 + 1;

Lox 내장 문법. 평가 결과를 stdout에 출력.

program→statement* EOF NEW
statement→exprStmt | printStmt NEW
exprStmt→expression ";"
printStmt→"print" expression ";"

4전역 변수

변수엔 두 동작 — 선언(이름↔값 묶기) + 접근(이름으로 꺼내기)

var beverage = "espresso";   // 선언 + 초기화
print beverage;                // 접근 → espresso
var empty;                     // 초기화 없으면 nil
varDecl→"var" IDENTIFIER ( "=" expression )? ";" NEW
primary→... | IDENTIFIER NEW
초기화를 생략하면 값은 nil. 에러가 아님 — 동적 타입 언어의 관례.

5declaration vs statement

"문장의 두 우선순위" — 선언이 되는 자리와 안 되는 자리

declaration→varDecl | statement NEW
statement→exprStmt | printStmt | block

넓은 자리 (declaration)

블록 안 · 최상위 → 선언 포함 뭐든 OK

var a = 1;  ✅

좁은 자리 (statement)

if/while 본문 → 선언 금지

if(c) var x=1; ❌
if (c) var x = 1; → x의 스코프가 애매. 그래서 막고, 블록 { }으로 감싸게 한다. declaration ⊇ statement

6Environment — 변수 저장소

본질은 HashMap<String, Object> 하나

class Environment {
  final Environment enclosing;          // 바깥(부모) 스코프
  final Map<String,Object> values = new HashMap<>();

  void   define(name, value)  // 새 변수 묶기 (재정의 허용)
  Object get(name)           // 값 꺼내기 (없으면 RuntimeError)
  void   assign(name, value)  // 기존 변수 수정 (없으면 RuntimeError)
}
변수는 토큰이 아니라 lexeme(문자열)로 저장 → 코드 곳곳의 a 토큰이 같은 바인딩을 가리킴.

8스코프 & 블록 — 환경 사슬

블록 { }마다 새 Environment, 그것이 바깥(enclosing)을 가리킴

var a = "global";
{ var a = "block";  print a; }  // block  (바깥 a를 가림 = shadowing)
print a;                          // global (블록 끝나면 복원)
Block Env 현재
a"block"
enclosing ──┐
→enclosing
Global Env 최상위
a"global"
enclosing = null
executeBlock()은 새 환경으로 실행하고 finally에서 복원 → 에러가 나도 환경이 새지 않음.

정리

핵심 3가지
