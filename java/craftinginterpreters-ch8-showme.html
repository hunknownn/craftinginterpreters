Crafting Interpreters · Chapter 8

문장이 상태를 만들고, 환경이 그 상태를 붙잡는다

8장은 Lox가 계산기에서 작은 프로그램 언어로 넘어가는 지점입니다. 표현식만 평가하던 인터프리터가 변수 선언, 대입, 블록 스코프를 실행하려면 “지금 어떤 이름이 어떤 값을 가리키는가”를 저장해야 합니다.

로드맵

표현식만으로는 부족 값을 계산할 수는 있지만, 계산 결과를 이름에 보관하거나 여러 줄 프로그램을 자연스럽게 실행하기 어렵다.
Stmt AST 추가 print, 표현식 문장, var, 블록을 별도 노드로 만들고 파서는 declaration부터 시작한다.
Environment Map<String, Object>와 바깥 환경 포인터로 이름 검색, 정의, 대입을 처리한다.
블록 스코프 중괄호에 들어가면 새 환경을 만들고, 끝나면 이전 환경으로 복구한다.

1. 왜 Stmt가 필요한가

문제 재현: 표현식 AST만 있을 때

7장까지의 인터프리터는 “값 하나를 평가한다”는 관점입니다. 그런데 아래 코드는 값을 계산하는 식과, 효과를 일으키는 문장이 섞여 있습니다.

var breakfast = "beignets"; print breakfast; breakfast = "espresso"; breakfast;
Stmt.Var 이름과 초기화 식을 받아 환경에 정의한다.
Stmt.Print 식을 평가한 뒤 사용자에게 출력한다.
Stmt.Expression 대입처럼 값보다 효과가 중요한 식도 한 줄 문장이 된다.

실행 로그

등장 원인

REPL 계산기를 넘어서 파일 단위 프로그램을 실행하려면 여러 문장을 순서대로 실행해야 한다.

해결 문제

표현식은 값을 돌려주지만, 변수 선언과 출력은 “효과”가 핵심이라 별도 AST 계층이 필요하다.

메커니즘

parse()가 List<Stmt>를 만들고 인터프리터는 각 문장을 방문한다.

대가

문법 계층이 늘고, 파서가 선언과 문장 복구까지 책임져야 한다.

2. Environment: 이름을 값으로 바꾸는 저장소

해결 메커니즘 시뮬레이션

define()은 현재 환경에 슬롯을 만들고, get()과 assign()은 현재 환경에서 시작해 바깥 환경으로 올라갑니다.

전역 Environmentenclosing = null
현재 → 바깥 검색
Object get(Token name) { if (values.containsKey(name.lexeme)) return values.get(name.lexeme); if (enclosing != null) return enclosing.get(name); throw undefined; }
블록 Environmentenclosing = global

환경 로그

선행 개념: 식별자

토큰의 lexeme가 맵 키가 된다. a라는 철자가 이름이다.

선행 개념: 런타임 오류

없는 이름을 읽거나 대입하면 파싱 오류가 아니라 실행 중 오류다.

트레이드오프

환경 체인은 단순하지만 깊은 스코프에서는 매번 선형 검색한다. 이후 resolver가 보완한다.

연결

함수 호출, 클로저, 클래스의 this도 결국 “어떤 환경을 붙잡는가”로 이어진다.

선택얻는 것잃는 것

// Parser: 프로그램은 문장 목록이다. List<Stmt> parse() { List<Stmt> statements = new ArrayList<>(); while (!isAtEnd()) statements.add(declaration()); return statements; } // var name (= initializer)? ; private Stmt varDeclaration() { ... }
// Interpreter: 블록은 새 환경에서 실행한다. public Void visitBlockStmt(Stmt.Block stmt) { executeBlock(stmt.statements, new Environment(environment)); return null; } void executeBlock(List<Stmt> statements, Environment env) { Environment previous = this.environment; try { this.environment = env; for (Stmt statement : statements) execute(statement); } finally { this.environment = previous; } }
